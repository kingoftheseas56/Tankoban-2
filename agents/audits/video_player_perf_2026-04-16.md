# Audit - Video player per-frame smoothness - 2026-04-16

By Agent 7 (Codex). For Agent 3 (Video player).
Reference comparison: mpv, IINA, QMPlay2, Microsoft DXGI/D3D11 documentation, Qt timer documentation.
Scope: This audit compares Tankoban's current decode, subtitle, D3D11 presentation, shared-memory, and clock pacing paths against the cited reference players. It does not assert root cause for the reported stutter; root-cause statements are limited to the hypotheses section for Agent 3 to validate. I did not build, run, or modify `src/`.

## Observed behavior (in our codebase)

1. The D3D11VA fast path exists, but subtitles disable it. In `video_decoder.cpp`, D3D11VA is enabled with frame threading disabled and the D3D11 presenter is initialized from codec parameters (`native_sidecar/src/video_decoder.cpp:247`, `native_sidecar/src/video_decoder.cpp:276`). The per-frame fast path copies a D3D11VA texture slice and then skips the CPU pipeline only when `zero_copy_active_ && d3d_gpu_copied && !sub_blend_needed` (`native_sidecar/src/video_decoder.cpp:407`, `native_sidecar/src/video_decoder.cpp:429`, `native_sidecar/src/video_decoder.cpp:431`). `sub_blend_needed` is true when subtitles are visible and an active subtitle stream is set (`native_sidecar/src/video_decoder.cpp:429`).

2. When the fast path is not taken on a D3D11VA frame, the decode thread transfers the hardware frame to CPU memory (`av_hwframe_transfer_data`), allocates/uses a BGRA output frame, converts high-bit-depth video through a 10-bit-to-8-bit `sws_scale` stage, then converts to BGRA with another `sws_scale` unless the GPU renderer succeeds (`native_sidecar/src/video_decoder.cpp:506`, `native_sidecar/src/video_decoder.cpp:510`, `native_sidecar/src/video_decoder.cpp:530`, `native_sidecar/src/video_decoder.cpp:548`, `native_sidecar/src/video_decoder.cpp:574`, `native_sidecar/src/video_decoder.cpp:624`, `native_sidecar/src/video_decoder.cpp:645`). For 1920x1080 BGRA this is about 8.29 MB per full-frame copy; for 1920x804 BGRA it is about 6.18 MB per full-frame copy.

3. Subtitle blending is synchronous from the decode thread's point of view. `video_decoder.cpp` calls `sub_renderer_->render_blend(...)` before `present_cpu(...)` and before writing the frame to SHM (`native_sidecar/src/video_decoder.cpp:722`, `native_sidecar/src/video_decoder.cpp:725`, `native_sidecar/src/video_decoder.cpp:730`, `native_sidecar/src/video_decoder.cpp:741`). `render_blend` posts a request under `render_mutex_`, notifies a render thread, then blocks on `render_done_cv_` until that render thread completes (`native_sidecar/src/subtitle_renderer.cpp:422`, `native_sidecar/src/subtitle_renderer.cpp:428`, `native_sidecar/src/subtitle_renderer.cpp:437`, `native_sidecar/src/subtitle_renderer.cpp:439`).

4. The subtitle render thread holds `subtitle_renderer::mutex_` while it calls `ass_render_frame` and while it CPU-blends the returned image list into the full BGRA frame (`native_sidecar/src/subtitle_renderer.cpp:151`, `native_sidecar/src/subtitle_renderer.cpp:162`, `native_sidecar/src/subtitle_renderer.cpp:175`). `process_packet`, called by the decode thread for subtitle packets, also holds the same mutex while it decodes PGS packets or calls `ass_process_chunk` for text/ASS packets (`native_sidecar/src/subtitle_renderer.cpp:275`, `native_sidecar/src/subtitle_renderer.cpp:278`, `native_sidecar/src/subtitle_renderer.cpp:291`, `native_sidecar/src/subtitle_renderer.cpp:370`, `native_sidecar/src/video_decoder.cpp:855`, `native_sidecar/src/video_decoder.cpp:863`). Initial subtitle preload also feeds packets through `process_packet` (`native_sidecar/src/main.cpp:610`, `native_sidecar/src/main.cpp:787`).

5. Subtitle blending is CPU pixel work. The libass path loops over every `ASS_Image`, then every bitmap row and column, and blends individual BGRA pixels (`native_sidecar/src/subtitle_renderer.cpp:490`, `native_sidecar/src/subtitle_renderer.cpp:502`, `native_sidecar/src/subtitle_renderer.cpp:509`). The PGS path similarly blends rect pixels in nested loops (`native_sidecar/src/subtitle_renderer.cpp:529`, `native_sidecar/src/subtitle_renderer.cpp:546`, `native_sidecar/src/subtitle_renderer.cpp:551`).

6. The CPU-present path uploads the full BGRA frame to the shared D3D11 texture with `UpdateSubresource`, then calls `Flush` every frame (`native_sidecar/src/d3d11_presenter.cpp:96`, `native_sidecar/src/d3d11_presenter.cpp:104`, `native_sidecar/src/d3d11_presenter.cpp:107`). The GPU-slice path copies the entire source subresource by passing a null source box to `CopySubresourceRegion`, then also calls `Flush` (`native_sidecar/src/d3d11_presenter.cpp:80`, `native_sidecar/src/d3d11_presenter.cpp:85`, `native_sidecar/src/d3d11_presenter.cpp:90`, `native_sidecar/src/d3d11_presenter.cpp:92`). The shared texture is BGRA8 and sized to the presenter's current width/height (`native_sidecar/src/d3d11_presenter.cpp:122`, `native_sidecar/src/d3d11_presenter.cpp:127`, `native_sidecar/src/d3d11_presenter.cpp:160`).

7. The sidecar still writes full CPU frames to SHM on the non-fast path after `present_cpu` (`native_sidecar/src/video_decoder.cpp:730`, `native_sidecar/src/video_decoder.cpp:741`). `FrameRingWriter::write_frame` copies the pixel payload with `memcpy` into the selected slot (`native_sidecar/src/ring_buffer.cpp:43`, `native_sidecar/src/ring_buffer.cpp:45`).

8. The main app render loop is timer-driven. `FrameCanvas` uses a precise `QTimer` with a 16 ms interval (`src/ui/player/FrameCanvas.cpp:36`, `src/ui/player/FrameCanvas.cpp:37`), clears, draws the textured quad, then calls `m_swapChain->Present(1, 0)` (`src/ui/player/FrameCanvas.cpp:523`, `src/ui/player/FrameCanvas.cpp:526`, `src/ui/player/FrameCanvas.cpp:528`). The swap chain is flip-discard with `BufferCount = 2`, but its flags are zero, so no frame-latency waitable object is requested (`src/ui/player/FrameCanvas.cpp:211`, `src/ui/player/FrameCanvas.cpp:213`, `src/ui/player/FrameCanvas.cpp:215`).

9. The `m_skipNextPresent` guard skips the next `Present()` after sustained timer/present overrun. It is armed when computed `frameLatencyMs` exceeds the threshold for enough ticks (`src/ui/player/FrameCanvas.cpp:480`, `src/ui/player/FrameCanvas.cpp:488`, `src/ui/player/FrameCanvas.cpp:540`, `src/ui/player/FrameCanvas.cpp:565`, `src/ui/player/FrameCanvas.cpp:567`). I could verify the guard exists and is timer-overrun driven; I could not verify from source alone whether it is currently firing during Hemanth's stutter sessions.

10. The SHM consumer does O(slot_count) scans on the render tick. `readBestForClock` walks every slot to find the newest valid frame at or before `clock + tolerance`, and `readLatest` walks every slot for fallback (`src/ui/player/ShmFrameReader.cpp:94`, `src/ui/player/ShmFrameReader.cpp:146`, `src/ui/player/FrameCanvas.cpp:1024`, `src/ui/player/FrameCanvas.cpp:1028`). The selected SHM frame is uploaded into a D3D11 texture with `UpdateSubresource` on the main render tick (`src/ui/player/FrameCanvas.cpp:1104`, `src/ui/player/FrameCanvas.cpp:1105`).

11. The A/V gate is synchronous in the video decode path. The video thread waits for the audio clock to start, then repeatedly samples `clock_->position_us()` and sleeps up to 20 ms until the frame PTS is within a 15 ms tolerance (`native_sidecar/src/video_decoder.cpp:672`, `native_sidecar/src/video_decoder.cpp:679`, `native_sidecar/src/video_decoder.cpp:697`, `native_sidecar/src/video_decoder.cpp:702`, `native_sidecar/src/video_decoder.cpp:706`). `AVSyncClock::position_us()` takes a mutex on every call (`native_sidecar/src/av_sync_clock.cpp:88`, `native_sidecar/src/av_sync_clock.cpp:90`). The audio clock is updated after blocking `Pa_WriteStream` returns (`native_sidecar/src/audio_decoder.cpp:337`, `native_sidecar/src/audio_decoder.cpp:632`, `native_sidecar/src/audio_decoder.cpp:648`).

12. The sidecar build script uses MinGW Makefiles and `-DCMAKE_BUILD_TYPE=Release`; I did not find sidecar-specific LTO, PGO, or `-march=native` flags in the build script (`native_sidecar/build.ps1:10`, `native_sidecar/build.ps1:24`, `native_sidecar/build.ps1:29`).

1. mpv queues decoded frames to the video output instead of making the decode-side path synchronously own the final display cadence. `player/video.c` pulls from the filter output and returns `VD_WAIT` when no frame is available (`C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:466`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:498`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:500`), then schedules and queues frames to the VO (`C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:1240`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:1270`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:1275`). ## Gaps (ranked P0 / P1 / P2)

**P0 (user-blocking or severely degrading):**

- Subtitle-active HEVC 10-bit playback falls off the D3D11VA zero-copy path and performs several full-frame CPU/GPU/SHM operations per frame. Observed: `sub_blend_needed` disables the fast path (`native_sidecar/src/video_decoder.cpp:429`, `native_sidecar/src/video_decoder.cpp:431`), then the path does GPU-to-CPU transfer, 10-bit conversion, BGRA conversion, CPU subtitle blend, `present_cpu`, and SHM write (`native_sidecar/src/video_decoder.cpp:510`, `native_sidecar/src/video_decoder.cpp:574`, `native_sidecar/src/video_decoder.cpp:645`, `native_sidecar/src/video_decoder.cpp:725`, `native_sidecar/src/video_decoder.cpp:731`, `native_sidecar/src/video_decoder.cpp:741`). Reference: mpv maps hardware frames and attaches subtitle overlays as GPU-side overlay textures (`C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\video\out\vo_gpu_next.c:681`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\video\out\vo_gpu_next.c:313`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\video\out\vo_gpu_next.c:337`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\video\out\vo_gpu_next.c:367`). Impact: this is the largest observed per-frame cost gap versus mpv for the exact 1920x1080 and 1920x804 HEVC 10-bit workload described.

- Present cadence is driven by a fixed 16 ms GUI timer and a post-hoc skip guard, not by a video-output frame queue with measured vsync timing. Observed: `QTimer` interval 16 ms and `Present(1, 0)` on each render tick (`src/ui/player/FrameCanvas.cpp:36`, `src/ui/player/FrameCanvas.cpp:37`, `src/ui/player/FrameCanvas.cpp:528`), with skip-next driven by measured timer overrun (`src/ui/player/FrameCanvas.cpp:540`, `src/ui/player/FrameCanvas.cpp:565`, `src/ui/player/FrameCanvas.cpp:567`). Reference: mpv calculates vsync count/error and queues frames to VO (`C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:837`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:855`, `C:\Users\Suprabha\Downloads\Video player reference\mpv-master\mpv-master\player\video.c:1275`), while Qt documents that timers can wake late under load. Impact: even if decode cost is acceptable, fixed-timer jitter plus single-skip behavior can produce visible uneven cadence.

- D3D11VA slice copy does not explicitly bound the source rectangle when decode textures are padded. Observed: `present_slice` passes `nullptr` for the source box and copies from the source subresource to a display-sized BGRA shared texture (`native_sidecar/src/d3d11_presenter.cpp:85`, `native_sidecar/src/d3d11_presenter.cpp:90`, `native_sidecar/src/d3d11_presenter.cpp:122`, `native_sidecar/src/d3d11_presenter.cpp:160`). Microsoft documents undefined behavior for invalid source/destination copy dimensions. Impact: this is specifically relevant to the reported 1920x804 / padded-pool case.
