## Hypothesized root causes (Agent 2 to validate)

- Hypothesis — Reader progress appears unreliable because the Qt port kept Tankoban-Max frontend progress calls keyed by absolute path while the Tankoban 2 contract and library UI expect SHA1 keys plus top-level book progress fields. Agent 2 to validate.
- Hypothesis — Settings appear to reset or behave inconsistently because the JavaScript settings calls still match Tankoban-Max's global settings API, while the C++ bridge stores settings by a `bookId` argument that the JS does not provide. Agent 2 to validate.
- Hypothesis — Visible rendering glitches are caused in part by showing the reader before Foliate's renderer reaches a stable pagination/layout state, because the local integration handles `load` and `relocate` but not renderer `stabilized`. Agent 2 to validate.
- Hypothesis — Font/theme/page-count jumps are caused in part by overlapping styling layers and the delayed second `setStyles()` call forcing pagination recalculation after initial layout. Agent 2 to validate.
- Hypothesis — Fullscreen and window resize glitches are caused in part by host resize only resizing the `QWebEngineView` without an explicit Foliate renderer relayout or stabilization wait. Agent 2 to validate.
- Hypothesis — Keyboard, page-turn, and selection glitches are caused in part by forwarding iframe key events into the parent document while the parent also owns global navigation shortcuts, creating focus-dependent dispatch paths. Agent 2 to validate.
- Hypothesis — Open-time stalls or flashes on larger EPUBs are caused in part by whole-file QWebChannel reads plus fallback `file://` fetch behavior instead of a single predictable file-loading path. Agent 2 to validate.
- Hypothesis — Listen/TTS appears broken or misleading because the frontend is an Edge TTS implementation but the Qt shim currently returns stubbed unavailable responses that mention Kokoro removal. Agent 2 to validate.
- Hypothesis — Some perceived Book Reader/Tankoban-Max mismatch is adapter mismatch rather than renderer mismatch: Tankoban-Max frontend modules were retained, but their expected Electron IPC domains were reimplemented with different C++ signatures and progress-key semantics. Agent 2 to validate.
