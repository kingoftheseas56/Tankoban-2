## Gaps (ranked P0 / P1 / P2)

### P0

- None found in this audit pass. Core open/read/navigate/continue behavior exists, and Congress 1/4 critical reader parity appears broadly represented in current code.

### P1

- Reading mode breadth and persistence lag manga/webtoon references. Current Qt has single-page, double-page, and scroll-strip only (`ComicReader.h:74-75`), while Mihon explicitly distinguishes LTR pager, RTL pager, vertical pager, webtoon, and continuous vertical (`ReadingMode.kt:14-79`). Current RTL is a transient double-page toggle and is not persisted in series settings (`ComicReader.cpp:1683-1689`, `ComicReader.cpp:2854-2913`). This is a visible miss for manga users who expect direction/mode to stick per series.
- Webtoon-long-strip is only partially covered by ScrollStripCanvas. Current continuous vertical stacks decoded pages with zero spacing and portrait-width scaling (`ScrollStripCanvas.cpp:151-198`, `ScrollStripCanvas.cpp:241-255`), but Mihon exposes webtoon-specific side padding, crop, split/rotate, and zoom-out preferences (`ReaderPreferences.kt:71-88`, `ReadingModePage.kt:140-205`). OpenComic also treats webtoon/scroll width adjustment specially in render layout (`scripts\reading.js:300-410`). Users coming from manga/webtoon readers would miss webtoon-specific tuning.
- Thumbnail sidebar/page grid is absent in-reader. Current has numeric go-to, scrub bar, and vertical strip thumb, but no visible page thumbnail rail or grid (`ComicReader.cpp:1805-1844`, `ComicReader.cpp:1322-1335`). YACReader's GoToFlow and OpenComic's left sidebar both use thumbnails for page seeking (`YACReader/goto_flow.cpp:29-55`, `YACReader/goto_flow.cpp:171-190`, `OpenComic/templates\reading.content.left.html:1-35`). Long 300-500 page files are therefore harder to visually navigate.
- Bookmark UX is functionally present but below reference-reader expectations. Current bookmarks are a page-index set toggled with `B`, surfaced only as a recent/sorted context-menu submenu (`ComicReader.cpp:2508-2518`, `ComicReader.cpp:2771-2782`). YACReader and OpenComic both provide bookmark surfaces with thumbnails and direct visual selection (`bookmarks_dialog.cpp:22-158`, `reading.elements.menus.collections.bookmarks.html:1-40`).
- Zoom/fit tooling is narrower than desktop comic-reader norms. Current fit choices omit actual-size/original-size and custom zoom, and zoom exists only in double-page mode through `Ctrl+wheel` at 100-260% (`ComicReader.h:74`, `ComicReader.cpp:1730-1740`). YACReader exposes FullRes, a zoom slider, zoom +/- and a magnifying glass (`configuration.h:21-26`, `main_window_viewer.cpp:295-350`, `main_window_viewer.cpp:406-429`); Mihon exposes original-size/smart-fit choices (`ReaderPreferences.kt:215-229`); OpenComic exposes global zoom settings (`settings.js:1449-1456`).
- Image processing is a Tankoban-Max divergence and an external parity gap. Current Qt exposes smoothing quality/memory-saver/gutter controls but no brightness, contrast, saturation, hue, invert/night, grayscale, auto-contrast, sharpness, or presets (`ComicReader.cpp:2466-2563`). Tankoban-Max had an image-filter panel and persisted canvas filters (`reader_view.html:76-158`, `mega_settings.js:354-384`, `mega_settings.js:605-671`); Mihon, OpenComic, YACReader, and MComix all expose at least some filter/enhancement controls (`ColorFilterPage.kt:23-125`, `reading.elements.menus.pages.filters.html:1-73`, `viewer.cpp:1222-1235`, `enhance_backend.py:10-37`).
- Settings UX is too scattered compared with references and Tankoban-Max. Current controls are split among toolbar buttons, right-click menus, keyboard-only toggles, and a keys overlay; persistence covers only a narrow set of series fields (`ComicReader.cpp:2466-2563`, `ComicReader.cpp:2675-2769`, `ComicReader.cpp:2854-2913`). Tankoban-Max had a consolidated Mega Settings surface (`reader_view.html:306-377`), Mihon has categorized reader settings (`ReadingModePage.kt:27-205`), and OpenComic has full settings/shortcut/tap-zone screens (`settings.js:712-875`, `settings.js:969-1060`).
- Mode switching is cycle-only in the primary reader chrome. Current `M`/mode button advances through three modes (`ComicReader.cpp:1476-1526`), while Mihon exposes direct icon selection for all reading modes (`ReadingModeSelectDialog.kt:29-76`). This is mainly a UX cost once more modes/settings exist, but it is already weaker than direct mode-picking references.
- Error handling collapses distinct failure modes into generic outcomes. Current open failures/no images become `No image pages found` or `File not found`, and per-page decode failures remove in-flight state without an observed page-level message (`ComicReader.cpp:728-734`, `DecodeTask.cpp:20-44`, `ComicReader.cpp:868-966`). YACReader at least has distinct loading and page-unavailable display states (`viewer.cpp:984-1002`). Password-protected RAR/CBR, corrupt archives, and missing mid-archive pages are likely indistinguishable to users.
- Archive discovery/entry UX is narrower than both our engine behavior and YACReader. Current SeriesView/ComicsPage reader-entry lists are CBZ-only (`SeriesView.cpp:25`, `SeriesView.cpp:581-589`, `ComicsPage.cpp:212-221`), while ArchiveReader has CBR/RAR paths (`ArchiveReader.cpp:26-38`, `ArchiveReader.cpp:84-138`) and YACReader discovers a broad archive set (`common/comic.cpp:63-99`). This is not an archive-internals finding; the user-visible gap is that supported/expected comic files can be absent from the reader entry points.

- Chapter/volume transition messaging is simpler than Mihon's boundary UX. Current end overlay covers next/replay/library (`ComicReader.cpp:480-547`), but there is no observed missing-volume/chapter-gap warning comparable to Mihon's transition warning card (`ChapterTransition.kt:173-199`).

Current reader entry appears to come from library/SeriesView/continue actions (`ComicReader.cpp:692-775`, `SeriesView.cpp:384-527`, `ComicsPage.cpp:617-708`), and no drop event wiring was observed in these classes. This is P2 because library-first flow exists, but desktop comic readers often support direct file opening as a convenience.
