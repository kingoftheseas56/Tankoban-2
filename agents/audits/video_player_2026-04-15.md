In scope: playback controls, keyboard shortcuts, seek bar UX, speed, loop/A-B repeat, frame step discoverability, video/audio/subtitle control UX, HUD/OSC behavior, context menu, window features, playlist/queue, opening/loading flows, advanced navigation, error handling, settings exposure, and performance visibility. Out of scope: D3D11 render pipeline internals, HDR/PQ/HLG/gamut/tonemapping engine work, audio drift correction, tanh amplification, DRC compressor, subtitle renderer internals, frame pacing/vsync smoothness, audio passthrough, stream-mode features, Videos library page UX, mobile patterns, and browser/web-player patterns.

## Observed behavior (in our codebase)

- The host opens local file paths through `VideoPlayer::openFile`, optionally accepts an explicit playlist/index, and otherwise builds a same-directory playlist from common video extensions (`src/ui/player/VideoPlayer.cpp:224-246`). Resume state is read from `LibraryVideos` and skipped near the end of a completed item (`src/ui/player/VideoPlayer.cpp:258-270`).
- Sidecar open commands can preserve HTTP/HTTPS URLs and carry `startSeconds` (`src/ui/player/SidecarProcess.cpp:114-122`), but the Qt player surface observed in `VideoPlayer` exposes local-file opening only, not an Open URL dialog or recent-file browser.
- Playback controls live in a bottom `m_controlBar` with an opaque black background (`src/ui/player/VideoPlayer.cpp:524-535`). Mouse movement shows the controls (`src/ui/player/VideoPlayer.cpp:1781-1785`), and the hide timer hides them unless playback is paused or seeking (`src/ui/player/VideoPlayer.cpp:1334-1351`).
- The seek bar supports click-to-jump, drag-to-seek, hover/drag time labels, and throttled live seek while dragging (`src/ui/player/VideoPlayer.cpp:584-641`, `src/ui/player/SeekSlider.cpp:74-98`). The hover preview is a time bubble only (`src/ui/player/VideoPlayer.cpp:1089-1096`).
- Chapter metadata is received from sidecar `media_info` into `m_chapters` (`src/ui/player/VideoPlayer.cpp:975-994`), and key handling supports previous/next chapter (`src/ui/player/VideoPlayer.cpp:1601-1705`), but the seek bar implementation does not draw chapter markers in the observed `SeekSlider`/`VideoPlayer` paths.
- Speed is a fixed preset menu: 0.5x, 0.75x, 0.9x, 1.0x, 1.25x, 1.5x, 2.0x (`src/ui/player/VideoPlayer.cpp:127-129`, `src/ui/player/VideoPlayer.cpp:714-750`). Keyboard speed up/down/reset walks those presets (`src/ui/player/VideoPlayer.cpp:1128-1150`).
- Frame stepping is exposed through key dispatch and the sidecar `frame_step` command (`src/ui/player/VideoPlayer.cpp:1601-1705`, `src/ui/player/SidecarProcess.cpp:140-147`). `frameStepped` events update the displayed time and slider (`src/ui/player/VideoPlayer.cpp:1076-1087`).
- Default shortcuts cover playback, seek, speed, volume, mute, fullscreen, deinterlace, audio/subtitle cycling, subtitle and audio delay, chapters, playlist, shortcuts overlay, and diagnostics (`src/ui/player/KeyBindings.cpp:3-54`). The bindings persist through `QSettings` (`src/ui/player/KeyBindings.cpp:74-93`).
- The shortcuts overlay is a static help panel (`src/ui/player/ShortcutsOverlay.cpp:9-33`). It omits some shipped shortcuts such as period/comma frame step, PageUp/PageDown chapter navigation, and Ctrl audio-delay keys, while listing bracket variants for speed that are not present in the default `KeyBindings` table.
- Windows media keys are handled for play/pause, stop, previous/next, volume, and mute (`src/ui/player/VideoPlayer.cpp:1928-1950`).
- Audio/subtitle track selection exists through `TrackPopover`, with Off entries and per-track rows (`src/ui/player/TrackPopover.cpp:276-329`). Track changes are merged from sidecar events, repopulated, and track preferences are restored by priority (`src/ui/player/VideoPlayer.cpp:1180-1210`, `src/ui/player/VideoPlayer.cpp:1398-1466`).
- Subtitle controls are split across `TrackPopover` and `SubtitleMenu`. `TrackPopover` exposes delay, size, margin, outline, color, and background opacity with a 300 ms debounce (`src/ui/player/TrackPopover.cpp:119-150`, `src/ui/player/TrackPopover.cpp:159-256`, `src/ui/player/TrackPopover.cpp:492-500`). `SubtitleMenu` exposes track choice, local subtitle loading, delay, vertical offset, and size (`src/ui/player/SubtitleMenu.cpp:62-153`).
- External subtitle loading is supported from the menu and sidecar URL/file path path (`src/ui/player/SubtitleMenu.cpp:247-263`, `src/ui/player/SidecarProcess.cpp:496-577`). The sidecar renderer handles embedded/libass/PGS/external composition, which is out of scope for this UX audit (`C:\Users\Suprabha\Desktop\TankobanQTGroundWork\native_sidecar\src\subtitle_renderer.cpp:72-100`, `C:\Users\Suprabha\Desktop\TankobanQTGroundWork\native_sidecar\src\subtitle_renderer.cpp:386-486`).
- One subtitle-off path in the context menu sends visibility false without changing track (`src/ui/player/VideoPlayer.cpp:1865-1886`), while `SubtitleMenu::applyChoice(Off)` calls `sendSetSubtitleTrack(-1)` (`src/ui/player/SubtitleMenu.cpp:247-263`), and that helper still emits a `set_tracks` command with sub id `"off"` (`src/ui/player/SidecarProcess.cpp:472-493`). The sidecar `set_tracks` handler parses nonempty subtitle ids with `std::stoi` (`C:\Users\Suprabha\Desktop\TankobanQTGroundWork\native_sidecar\src\main.cpp:802-884`).
- The volume surface supports 0-200 percent, sends a 0.0-2.0 scalar to sidecar, and displays values above 100 percent in the HUD (`src/ui/player/VideoPlayer.cpp:1301-1315`, `src/ui/player/VolumeHud.cpp:63-70`, `src/ui/player/VolumeHud.cpp:142-166`). Sidecar volume clamps to 0.0-2.0 and applies tanh soft clipping above unity, which is out of scope for re-audit (`C:\Users\Suprabha\Desktop\TankobanQTGroundWork\native_sidecar\src\volume_control.h:8-12`, `C:\Users\Suprabha\Desktop\TankobanQTGroundWork\native_sidecar\src\audio_decoder.cpp:540-560`).
- Equalizer and normalization are exposed from `EqualizerPopover`; the host maps EQ bands to FFmpeg equalizer filters and can combine normalization with EQ (`src/ui/player/VideoPlayer.cpp:901-913`).
- Video filters expose deinterlace, interpolation, normalization, and shader brightness/contrast/saturation; gamma is passed as neutral 1.0 in the current host path (`src/ui/player/VideoPlayer.cpp:930-940`). FrameCanvas has a gamma parameter and fixed-ratio aspect override support (`src/ui/player/FrameCanvas.h:36-39`, `src/ui/player/FrameCanvas.h:287-290`).
- The context menu covers play, mute, speed presets, aspect ratio presets, fullscreen, audio/subtitle tracks, external subtitles, subtitle settings, deinterlace, normalization, track popover, playlist, and back (`src/ui/player/VideoPlayer.cpp:1800-1925`).
- PlaylistDrawer is a 320 px drawer with a file list and one Auto-advance checkbox (`src/ui/player/PlaylistDrawer.cpp:17-29`, `src/ui/player/PlaylistDrawer.cpp:63-99`). EOF auto-advances only when a playlist exists, the current item is not the last item, and auto-advance is enabled (`src/ui/player/VideoPlayer.cpp:403-414`).
- The player supports fullscreen toggling and double-click fullscreen (`src/ui/player/VideoPlayer.cpp:1113-1117`, `src/ui/player/VideoPlayer.cpp:1787-1790`). I did not find player code paths for picture-in-picture, always-on-top, snapshot export, A-B repeat, explicit in-video bookmarks, shuffle, repeat-one, repeat-all, rotate, mirror, or subtitle text search in the audited files.
- Errors from sidecar are surfaced as toast messages (`src/ui/player/VideoPlayer.cpp:417-420`). Sidecar emits recoverable decode errors without changing playback state (`C:\Users\Suprabha\Desktop\TankobanQTGroundWork\native_sidecar\src\main.cpp:448-461`).
- Performance diagnostics are mostly hidden: Ctrl+Shift+V can write a `_vsync_timing.csv` timing log (`src/ui/player/VideoPlayer.cpp:1716-1743`), and `FrameCanvas` exposes vsync logging hooks (`src/ui/player/FrameCanvas.h:82-83`). I did not find an in-player stats overlay, codec badge, bitrate/FPS display, or buffer panel in the audited UI files.
